Data Engineering Technical Exam

A long-form assessment designed to validate architecture, cloud services, orchestration, SQL, Python, and fundamental data engineering skills.

Exam Instructions

This assessment is structured as a multi-part exam. Answer each question clearly and concisely. Use diagrams, bullet points, or examples where appropriate.

  • Section 1 focuses on infrastructure, cloud and platform design.
  • Section 2 covers ETL, orchestration, and core data tooling.
  • Section 3 tests SQL, data modeling, and query logic.
  • Section 4 evaluates Python engineering and data pipeline implementation.
  • Section 5 probes operational concepts, security, monitoring, and cost optimization.

Exam Blueprint

Cloud Infrastructure

AWS, Azure, GCP, networking, storage, and compute choices.

Data Platforms

Apache Spark, Airflow, Alteryx, dbt, Kafka, and modern ETL orchestration.

Analytics & Fundamentals

SQL, normalization, data modeling, reliability, monitoring, and governance.

Core Question Set

Section 1
Cloud Architecture Design
Describe a resilient multi-region data ingestion architecture for a global e-commerce application. Include AWS, Azure, or GCP components and explain how you would minimize latency, ensure data durability, and control cost.
Section 1
Storage & Compute Trade-offs
Compare AWS S3, Azure Data Lake Storage, and GCS for analytics workloads. When would you choose object storage over block storage and how do you ensure efficient columnar query performance?
Section 1
Cloud Networking
Explain VPC/subnet design for a secure data platform. Include private endpoints, firewall rules, and how to safely expose APIs and ingestion endpoints.
Section 2
Apache Spark Pipeline
Design a Spark pipeline that reads raw JSON events, performs schema evolution handling, deduplicates records, and writes partitioned Parquet output.
Section 2
Airflow Orchestration
Describe how to build a robust Airflow DAG with retries, SLA monitoring, and backfills for a nightly batch pipeline. Explain how you would parameterize the DAG and enforce task dependencies.
Section 2
Alteryx and No-Code ETL
Outline the strengths and limitations of Alteryx when used for enterprise data workflows. When is it a good fit, and when should you prefer code-first orchestration?
Section 3
SQL Query Logic
Write SQL that finds the top 3 customers by revenue for the last 90 days, excluding refunds and returning the customer segment and average order frequency.
Section 3
Data Modeling
Compare star schema and normalized relational models. Give examples of when each model is preferable in a reporting pipeline and how you would implement slowly changing dimensions.
Section 3
SQL Performance
Explain partition pruning, indexing strategy, and cost-based optimization for a data warehouse query. How do you detect and fix a query that performs full scans on a very large fact table?
Section 4
Python Data Engineering
Describe how you would implement a Python data ingestion job using pandas, PySpark, or dbt. Include error handling, retries, and how to manage schema validation.
Section 4
API & Streaming
Design a Python microservice that ingests streaming data from Kafka and writes normalized records to a data lake. Explain consumer groups, offsets, and exactly-once semantics.
Section 4
Testing & Quality
What automated validation checks would you apply to data pipelines? Include schema checks, row counts, null thresholds, and integration tests for Python processing code.
Section 5
Security & Governance
Explain role-based access control, encryption-at-rest, encryption-in-transit, and audit logging for a cloud data platform. How do you enforce least privilege across AWS, Azure, and GCP?
Section 5
Monitoring & Observability
Describe monitoring metrics and alert rules for a data pipeline. Include SLA tracking, pipeline health dashboards, and how to troubleshoot downstream data quality incidents.
Section 5
Cost Optimization
What strategies would you use to control costs in cloud analytics workloads? Cover storage lifecycle, compute scaling, reserved instances, and query cost management.

Bonus Deep-Dive Questions

Data Mesh & Hybrid Architecture

Explain the principles of data mesh and how you would apply them across teams, domains, and shared infrastructure in a large enterprise.

Metadata & Catalog

Describe the role of a metadata catalog, lineage tracking, and business glossary in a governed data platform. What tools or services would you choose?
